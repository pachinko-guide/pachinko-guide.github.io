# 更新ガイド（データの増やし方）

このサイトは **静的HTML/CSS/JS のみ**で動きます。サーバーやビルドは不要です。
`index.html` をブラウザでダブルクリックして開けば、そのまま検索できます。

---

## 1. データはどこにある？

すべて `data/` フォルダの中の JavaScript ファイルです。

| ファイル | 中身 |
|---|---|
| `data/qa.js` | 広告宣伝ガイドライン Q&A（補足資料）|
| `data/cases_05.js` | 是正勧告事例 第5集（2025年1〜6月）|
| `data/cases_06.js` | 是正勧告事例 第6集（2025年7〜12月）|

各ファイルは、こういう「レコード」の集まりです👇

```js
{
  id: "c6-21",                      // 重複しない名前（何でもOK）
  type: "case",                     // "qa" か "case"
  source: "是正勧告事例 第6集",       // 出典（フィルタに自動で出ます）
  period: "2025年7月～12月",         // 対象期間（事例のみ。Q&Aは不要）
  category: "禁止とする時差開店を表示した事例", // 分類（自動でプルダウンに出ます）
  no: "㉑",                         // 問番号や事例番号
  title: "見出し（質問文 / 事例の見出し）",
  body:  "本文（回答 / 事例の詳しい説明）",
  refs:  ["ガイドライン5.(9)"],       // 参照条文（任意・配列）
  keywords: ["リニューアル", "用語使用"] // 検索ヒット用キーワード（任意・配列）
}
```

> `keywords` は画面には出ませんが、**検索にヒットさせたい言葉**を入れておくと便利です。
> （例：略称・別名・関連機種名 など）

## 2. 既存ファイルに1件足したいとき

例：`data/cases_06.js` を開いて、`]);` の直前に `,` と新しい `{ ... }` を追記して保存するだけ。

```js
  },
  {
    id: "c6-22",
    type: "case",
    source: "是正勧告事例 第6集",
    period: "2025年7月～12月",
    category: "禁止とする時差開店を表示した事例",
    no: "㉒",
    title: "（追加した事例の見出し）",
    body:  "（説明）",
    refs:  ["ガイドライン5.(5)"],
    keywords: ["時差開店"]
  }
]);
```

## 3. 新しい事例集／資料がまるごと増えたとき

1. `data/` に新しいファイルを作る（例：`data/cases_07.js`）。
   - 中身は既存ファイルをコピーして書き換えるのが簡単です。
   - 先頭は必ず `window.RECORDS = (window.RECORDS || []).concat([` で始め、最後は `]);` で閉じる。
2. `index.html` の下のほうにある読み込み行に **1行追加**：

```html
  <script src="data/qa.js"></script>
  <script src="data/cases_05.js"></script>
  <script src="data/cases_06.js"></script>
  <script src="data/cases_07.js"></script>  <!-- ← この行を追加 -->
  <script src="assets/app.js"></script>
```

これだけで、検索・フィルタ（種別／出典／分類）に自動で反映されます。
`assets/app.js` は触らなくてOKです。

## 3.5 事例の「画像」について

是正勧告事例カードには、原典PDFの該当ページを画像化したもの（`assets/images/*.jpg`）が表示されます。
紐付けは `data/images.js`（`window.CASE_IMAGES = { 事例id: [画像パス] }`）で管理しています。

- この `images.js` と `assets/images/` は **元PDFから自動生成**したものです（手編集不要）。
- 新しい事例集PDFを追加するときは、Claudeに「画像も入れて」と頼めば、
  同じ方式でページ画像を書き出し `images.js` に追記します。
- 個別レコードに `images: ["assets/images/xxx.jpg"]` を直接書いてもOK（そちらが優先されます）。

## 4. 元データ（PDF）

`元データ/` フォルダに、データの根拠となった公式PDFを保存しています。
迷ったときは必ず原典を確認してください。

---

## 5. クロード（Claude Code）に頼むとき

「新しい是正勧告事例のPDFを `data/` に追加して」と、PDFを渡して頼めば
この形式に沿って自動で追記します。
